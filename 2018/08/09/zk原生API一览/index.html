zk原生API一览

maven依赖

创建会话

zk客户端通过创建Zookeeper实例来连接Zookeeper服务器，org.apache.zookeeper.Zookeeper包含四个构造函数

参数说明

  • connectString:这是zk的服务器列表，形式为ip:port,ip:port,...ip:port,也可以通过ip:port,ip:port,...ip:port/example_node这样来表示客户端连上服务器后的跟布鲁，以后的各种操作都会基于该根目录，例如客户端操作/son1/son1-1则会指向/example_node/son1/son1-1,这个目录叫做Chroot
  • sessionTimeout:会话超时时间，单位为毫秒的整型值，zk客户端和服务端会通过心跳机制来维持会话的有效性，如果在sessionTimeout时间内没有收到心跳检测，则会话失效
  • watcher:客户端需要传入Watcher接口的实现类作为默认的Watcher事件通知处理器，也可以设置为null
  • canBeReadOnly:表示当前会话是否支持只读模式，有时候集群中一个机器和其他过半数机器失去联系，如果设置了只读模式，则当前机器可以提供读服务，不能写，如果没有设置只读模式，则不能提供读/写服务
  • sessionId/sessionPasswd:会话id/密钥，这两个参数可以唯一确定一个会话，客户端可以通过这两个参数实现会话复用/会话恢复。当第一次连接到服务器可以获得这两个参数，在下次连接时传入Zookeeper的构造函数即可

注意: 会话建立的过程是异步的，构造方法在处理完客户端的初始化工作后立即返回，此时很可能并没建立一个可用的会话，会话的状态只是处于CONNECTING状态。当会话真正建立完毕，zk服务端会像对应的客户端发送一个事件通知，此时构造函数中Watcher的作用就会体现，刻划断获取这个通知之后会话真正建立。

创建节点

创建节点有如下几个重载方法

参数说明

  • path:要创建的数据节点路径
  • data[]:字节数组，节点创建后的初始内容
  • acl:ACL策略，权限控制
  • createMode:节点类型，有四种，如下:
    • 持久PERSISTENT:会话断开也不会销毁
    • 持久顺序PERSISTENT_SEQUENTIAL:会话断开也不会销毁且其后会跟随整型zk全局id
    • 临时EPHEMERAL:会话断开则节点销毁
    • 临时顺序EPHEMERAL_SEQUENTIAL:会话断开则节点销毁,其后会跟随整型zk全局id
  • cb:异步回调函数，实现StringCallback接口，重写processResult方法，服务端节点创建完毕后会自动调用该方法，其中processResult方法参数如下
    • rc:ResultCode，服务端响应码，通过rc可以知道api调用的结果，常见响应码：
      • 0:OK,接口调用成功
      • -4:ConnectionLoss,客户端和服务段断开连接
      • -110:NodeExists指定节点已存在
      • -112:SessionExpired,会话过期
  • ctx:用于参数传递，可在回调方法执行时使用

注意: zookeeper不支持递归创建节点，且同名节点如果存在会抛NodeExistsException异常，zookeeper不提供序列化，要开发人员自行序列化/反序列化

删除节点

删除节点的重载方法如下:

参数说明:

  • path:要删除的节点路径
  • version:指定节点的数据版本，即表明本次删除操作是针对该数据版本进行
  • cb:异步删除之后的回调函数，实现该接口，重写public void processResult(int rc, String path, Object ctx)方法，几个参数含义与上边异步创建节点的回调函数参数类似
  • ctx:用于参数传递，可在回调方法执行时使用

查询子节点列表

参数说明

  • path:指定数据节点的节点路径，即API调用的目的是获取该节点的子节点列表
  • watcher:监听的watcher,在本次子节点获取之后，子节点列表发生变更的话那么就会向客户端发送通知，该参数可以传null。如果客户端在获得指定节点的子节点列表后，还需要订阅这个子节点列表的变化通知，那么就可以通过注册watcher来实现。当有子节点被添加或者删除时，服务端会向客户端发送NodeChildrenChanged类型的事件通知。在通知中，不包含最新的节点列表，客户端要自己获取
  • watch:表明是否需要注册watcher，如果为true，那么zookeeper就会使用创建zookeeper会话时传入的watcher,如果为false,则不需要注册watcher
  • cb:异步回调函数，ChildrenCallback与Children2Callback的区别是Children2Callback会在processResult方法中待会新的stat对象，而ChildrenCallback不会
  • ctx:上下文传递的参数
  • stat:指定数据节点的状态信息，在方法中传入旧的状态stat,该stat会被来自来自服务端响应的新stat对象替换，当我们需要最新的子节点状态信息时可以传入该参数

注意: 该方法只能获得当前节点的子节点，不能递归再往下会的更多层次

查询节点数据

参数说明

  • path:要获得数据的节点路径
  • watcher:注册的watcher,注册之后节点状态发生变更，就会向客户端发送NodeDataChanged的事件通知，该参数可以传入null
  • stat:数据节点的状态信息，跟getChildren方法的参数一致
  • watch:同getChildren
  • cb:同getChildren
  • ctx:同getChildren

注意: 节点的数据内容或者节点的数据版本变化，都被看做zookeeper节点的变化

更新数据

参数说明

  • path:数据节点路径
  • data[]:要设置成的数据，覆盖旧数据
  • version:指定节点的数据版本，即表明本次操作是针对该数据版本机型，
  • cb:异步回调函数
  • ctx:传递的参数

注意: setData函数有version参数，而getData却没有对应的参数，是应为在分布式的环境下，在更新某个节点数据时，多个线程用cas操作进行更新，version就是cas里面的预期值，针对该版本更新数据。version是从0开始，每次加一，-1表示一个标识符，表示对数据最新版本进行更新操作，如果zk节点的数据更新操作没有原子性要求，则可以使用-1

检测节点存在

  • path:数据节点路径
  • watcher:用于监听节点创建/删除/更新的事件
  • watch:是否复用默认的watcher
  • cb:异步回调函数
  • ctx:传递参数

注意: 对于指定节点的子节点的各种变换，不会发出事件通知

权限控制

参数说明

  • scheme:权限控制模式，world/auth/digest/ip/super
  • auth:集体权限信息

ZooDefs.Ids说明

  • OPEN_ACL_UNSAFE ：完全开放。 事实上这里是采用了world验证模式，由于每个zk连接都有world验证模式，所以znode在设置了 OPEN_ACL_UNSAFE 时，是对所有的连接开放。
  • CREATOR_ALL_ACL ：给创建该znode连接所有权限。 事实上这里是采用了auth验证模式，使用sessionID做验证。所以设置了 CREATOR_ALL_ACL 时，创建该znode的连接可以对该znode做任何修改。
  • READ_ACL_UNSAFE ：所有的客户端都可读。 事实上这里是采用了world验证模式，由于每个zk连接都有world验证模式，所以znode在设置了READ_ACL_UNSAFE时，所有的连接都可以读该znode。

注意: 当我们对一个节点添加权限信息后，依然可以自由删除这个节点，但对于这个节点的子节点，就必须使用相应的权限信息才能删除。
